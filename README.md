## 🧩 How I Build an AI Feature

<p align="center"><i>A simple look at how I make AI give accurate, trustworthy answers — step by step.</i></p>

```mermaid
flowchart LR
    A([Your question]) --> B[Find the most<br/>relevant information]
    B --> C[Rank it so only the<br/>best evidence is used]
    C --> D((AI reasons<br/>over the facts))
    D -->|can use tools| E[/Take actions ·<br/>fetch live data/]
    E --> D
    D --> F([Clear, accurate<br/>answer])
    G{{Quality checks}} -.keeps it honest.-> D

    classDef ai fill:#00A2FF,stroke:#0077cc,color:#fff;
    classDef step fill:#1f2937,stroke:#374151,color:#e5e7eb;
    classDef io fill:#111827,stroke:#00A2FF,color:#fff;
    class D ai;
    class B,C,E,G step;
    class A,F io;
```

<details>
<summary><b>🔧 The technical version (for fellow engineers)</b></summary>

<br/>

Retrieval-augmented generation done properly: **hybrid search** (semantic embeddings on
**pgvector** + keyword/BM25), fused with **Reciprocal Rank Fusion**, then **cross-encoder
re-ranking** so the model only sees the strongest evidence. On top: **function-calling
agents** and a **from-scratch MCP server**, orchestrated with **LangChain / LangGraph**
(StateGraph, persistence, human-in-the-loop), responses **streamed over SSE**, and the whole
thing wrapped in an **eval harness** so quality is a number, not a vibe.
Models: Groq · Gemini · Llama 3.3.

</details>
